# 新增仓库维护流程 Skill

日期: 2026-06-03

## 背景

本仓库维护时形成了一个固定习惯: 先制定计划，确认计划后实施，实施完成后记录维护历史。为了让后续新的 AI agent 也能遵循这个习惯，需要将其沉淀为仓库内可参考的 skill。

## 本次更新

- 新增 `.ai/skills/repository-maintenance-cycle/SKILL.md`。
- 新增 `.ai/skills/repository-maintenance-cycle/agents/openai.yaml`。
- 在 skill 中明确:
  - `.tmp/` 用于本地计划草稿，不提交。
  - `.maintenance/logs/` 用于已完成维护工作的仓库历史，应提交。
  - 非平凡变更应先计划，再实施，再验证，再写日志。
  - `public/` 是 Hugo 构建产物，不作为源码维护。
