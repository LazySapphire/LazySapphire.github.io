# 仓库源码化与构建流程标准化

日期: 2026-06-03

## 背景

仓库原本主要存放 Hugo 生成后的静态产物，包括根目录 HTML、RSS、sitemap、压缩 CSS/JS、tags/categories/posts/resume 等发布文件。后续维护目标调整为:

- 作为个人静态网站展示项目履历和背景。
- 同时作为个人笔记网站长期维护。
- 可在任意电脑 clone 后继续编辑、提交、部署。
- 不再复刻旧公开文档、旧工作记录和旧简历内容。

## 本次更新

- 建立 Hugo 源码结构:
  - `config/_default/`
  - `content/`
  - `layouts/`
  - `assets/`
  - `static/`
  - `scripts/`
- 使用自包含 Hugo templates，不依赖外部主题下载。
- 新增首页、About、Resume、Blog、Projects、Tools 的最小内容入口。
- 将通用图片、favicon、manifest 迁入 `static/`。
- 删除旧的生成后发布产物和旧公开内容目录。
- 将 GitHub Pages workflow 改为:
  - 安装 Hugo 0.125.4
  - 构建 `public/`
  - 运行生产产物检查
  - 只上传 `public/` 到 GitHub Pages
- 新增 `scripts/check-production-artifacts.sh`，阻止本地开发地址和旧模板痕迹进入部署产物。
- 新增仓库维护 README，说明源码目录、预览和内容维护流程。

## 决策

- `public/` 不提交。
- `public/` 只作为 Hugo 构建输出和 GitHub Pages 部署 artifact。
- 需要长期维护的内容应放在源码目录中:
  - 内容: `content/`
  - 模板: `layouts/`
  - 样式脚本: `assets/`
  - 静态资源: `static/`
- `.tmp/` 继续作为本地临时计划目录，不作为仓库历史提交。
- 已完成的计划整理为 `.maintenance/logs/` 下的维护日志。

## 验证

- 本地使用 Hugo 0.125.4 构建通过。
- `scripts/check-production-artifacts.sh public` 通过。
- `public/sitemap.xml` 和 RSS 使用正式域名 `https://lazysapphire.github.io/`。
- 本地 Hugo 预览服务可访问 `http://127.0.0.1:1313/`。

## 后续

- 后续真实内容替换时，从 `content/` 新增或更新 Markdown。
- 新文章建议使用英文小写短横线 slug。
- 如需要记录新的仓库维护动作，继续在 `.maintenance/logs/` 下新增日志。
