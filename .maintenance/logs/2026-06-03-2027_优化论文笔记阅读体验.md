# 优化论文笔记阅读体验

日期: 2026-06-03

## 背景

发布 Heracles 论文笔记后，网页检查发现三处阅读体验需要优化：正文中的 inline 变量仍是普通代码样式，Papers 列表缺少每篇论文的代表图，论文正文宽度在桌面阅读时略窄。

## 实施内容

- 将 Heracles 笔记正文中的数学变量和紧凑表达式从 code span 改为 inline LaTeX。
- 为 Heracles 笔记增加 `cover: "fig2_framework.png"` front matter。
- 更新列表模板，使带 `cover` 的页面在栏目列表中显示缩略图。
- 使用 Hugo image pipeline 为列表图生成较小缩略图，避免列表页直接加载原始大图。
- 调整 Papers 论文正文宽度，将 `.paper-note` 放宽到 1020px，正文 `.prose` 放宽到 940px。
- 保持论文正文容器在现有 `.site-main` 中左对齐，使阅读区域略偏左且不铺满屏幕。
- 更新 paper-note publishing skill，加入 inline math、cover thumbnail 和列表图约定。

## 关键决策

- 变量如 `$m_t$`、`$p_t$`、`$N_{\mathrm{exec}} = 2$` 使用 inline math；普通术语仍保持普通文本。
- 论文列表图采用 `cover` front matter 显式指定，而不是自动拿正文第一张图。
- 列表图使用 `Fit "360x225"` 生成缩略图，并用 `object-fit: contain` 避免裁掉论文示意图关键内容。

## 后续

- 后续论文笔记发布时，需要在 front matter 中设置合适的 `cover`。
- 如果后续论文列表图较多，可继续优化缩略图尺寸、懒加载或列表布局密度。
