# Feature Maintenance - 2026-W23

## 2026-06-03 - 仓库源码化与构建流程标准化

背景:

- 仓库从维护生成后的静态产物，调整为维护 Hugo 源码。
- 目标是让个人主页、项目履历和笔记内容都能通过源码长期维护，并可在其他电脑 clone 后复现。

实施内容:

- 建立 `config/`、`content/`、`layouts/`、`assets/`、`static/`、`scripts/` 等 Hugo 源码结构。
- 将旧生成产物从源码层移除，明确 `public/` 为构建输出。
- 新增最小内容入口：首页、About、Resume、Blog、Projects、Tools。
- 调整 GitHub Pages workflow：安装 Hugo、构建 `public/`、运行生产产物检查、上传 artifact。
- 新增 `scripts/check-production-artifacts.sh` 和维护 README。

验证:

- Hugo 构建通过。
- `scripts/check-production-artifacts.sh public` 通过。
- 本地预览服务可访问。

## 2026-06-03 - 新增仓库维护流程 Skill

背景:

- 仓库维护形成了计划、实施、验证、记录的固定习惯，需要沉淀给后续 AI agent。

- 新增 `.ai/skills/repository-maintenance-cycle/SKILL.md`。
- 新增对应 `agents/openai.yaml`。
- 明确 `.tmp/`、`.maintenance/logs/`、`public/` 和源码目录职责。

验证:

- 使用 skill-creator quick validation 校验通过。

## 2026-06-03 - 新增 AI 技能选择引导

背景:
